Installation

Install with Composer

  1. Installation requires explicit repositories:

    composer config repositories.d420c4664ef4513f6bec1e47bd4c5144 github https://github.com/aklump/easy-perms
composer config repositories.9024aba2baa64b08169c0ca94a832bc7 composer https://packages.intheloftstudios.com
composer config repositories.82ef065c6a1b8a21d8252f14c5f90789 path /Users/aklump/Code/Packages/php/fixture-framework
composer config repositories.619ded761a01c386b8878bc732c4cc2a github https://github.com/aklump/directio

  2. Require the latest stable version:

    composer require --dev aklump/directio:^0.0

  3. ... or require the dev channel:

    composer config minimum-stability dev
composer require --dev aklump/directio:@dev

Sandbox Install with Composer

A sandboxed installation separates this project, its dependencies, and even the PHP version from your main application. The project is installed in an isolated, hidden subdirectory in the root of your project. This is particularly useful if you have conflicts with your application's dependencies or if you need to use a different PHP version. It functions similarly to asdf shims, providing a project-specific environment for the tool.

Install in your Project

  1. Create the isolated directory (the sandbox):

    mkdir -p .directio
cd .directio
composer init -n --name=sandboxed/directio
composer config version dev-main
echo "vendor/" > .gitignore
touch README.md
echo "Learn more: https://github.com/aklump/directio\n" >> README.md
echo "Execute with: .directio/vendor/bin/directio\n" >> README.md

  2. Installation requires explicit repositories:

    composer config repositories.d420c4664ef4513f6bec1e47bd4c5144 github https://github.com/aklump/easy-perms
composer config repositories.9024aba2baa64b08169c0ca94a832bc7 composer https://packages.intheloftstudios.com
composer config repositories.82ef065c6a1b8a21d8252f14c5f90789 path /Users/aklump/Code/Packages/php/fixture-framework
composer config repositories.619ded761a01c386b8878bc732c4cc2a github https://github.com/aklump/directio

  3. Require the latest stable version:

    composer require --dev aklump/directio:^0.0

  4. ... or require the dev channel:

    composer config minimum-stability dev
composer require --dev aklump/directio:@dev

Optional: Create the Shim Script

To override a global binary, or to get around having to call the full path to your sandboxed binary, you can create a shim script. That allows you to call directio from anywhere in your project.

  1. Create bin/shim-composer.sh in the root of you application from https://gist.github.com/aklump/83e1f3be34d20fa61c6d9293a8c592d9. If you have already done so for another sandboxed package, you can skip this step and reuse that existing script.

    mkdir -p bin
touch bin/shim-composer.sh
chmod u+x bin/shim-composer.sh

How it Works

Shimming redirects shell commands to the sandboxed version by prepending the sandbox's bin directory to your $PATH.

$ which directio
/Users/aklump/Code/Packages/cli/directio/app/directio
$ . bin/shim-composer.sh directio
😎 directio shimmed to /Users/aklump/temp/php81/.directio/vendor/bin/directio
$ echo $PATH
/Users/aklump/temp/php81/.directio/vendor/bin:/Applicati...
$ which directio
/Users/aklump/temp/php81/.directio/vendor/bin/directio

Sandbox Usage

  1. At the start of a new terminal window call the shim script: . bin/shim-composer.sh directio. Note: This change lasts only for the current terminal session.
  2. Now simply call directio and it will point to .directio/vendor/bin/directio

Sandbox Updates

Updating your sandboxed installation requires an extra step beyond your main app composer update.

cd .directio && composer update
